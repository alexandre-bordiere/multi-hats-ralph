# Styling Guide

## Overview

Styling covers the naming, ordering, and structural conventions used across the codebase. The goal is a predictable layout so that any developer (or agent) can locate and extend code without guessing.

## Principles

- **Consistency throughout the codebase**: Styling must be consistent across the entire codebase. If a convention exists (explicit in this guide or implicit in neighbouring files), new code must follow it. When in doubt, mirror the closest existing sibling rather than introducing a new style.
- **Framework conventions first**: Styling must follow the conventions of the project's framework. Directory layout, class naming, namespacing, and file placement should match what a standard application using that framework would use. Do not invent bespoke structures when a framework-native one exists.
