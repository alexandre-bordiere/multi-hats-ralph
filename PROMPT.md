# Setup

Ask the user focused questions to understand their stack and conventions, then update `specs/guide/coding.md`, `specs/guide/styling.md`, and `specs/guide/testing.md` to reflect their answers.

The guides ship with sensible defaults — only update what the user's answers actually change. Cover:

- Language, framework, and any key libraries
- Coding conventions and architecture patterns
- Authentication, authorisation, and input validation approach
- Test runner, test types, and how tests are structured and run
- Directory layout and naming conventions
- Linting command (if applicable) — to fill in the `Usage` section of `specs/guide/styling.md`
- Testing command — to fill in the `Usage` section of `specs/guide/testing.md`
